## 系统概述

| 项目信息 | 详情 |
|---|---|
| 项目名称 | 需求管理平台（Requirement Management Platform） |
| 前端技术 | 纯 HTML + CSS + Vanilla JS（无框架） |
| 后端技术 | Python + FastAPI + SQLAlchemy |
| 数据库 | PostgreSQL（生产）/ SQLite（本地） |
| AI 引擎 | DeepSeek API（需求归并） |
| 适用角色 | PM/管理员、大区人员、SaaS 厂商人员 |

**三种角色视图：**
- **PM/管理员**：`需求看板.html` + `所有需求.html` + `大区管理.html` + `SaaS厂商管理.html`
- **大区人员**：`需求提报.html` + `我的需求追踪.html`
- **SaaS 厂商人员**：`SaaS需求提交.html` + `SaaS需求追踪.html`

## 页面功能详情

### 页面 1：需求看板（`需求看板.html`）

> 文件：`需求看板.html`（原 `dashboard.html`）

#### 1.1 侧边栏（Sidebar）

| 元素 | 类型 | 功能说明 |
|---|---|---|
| `需求管理平台` | 标题 | 系统名称，始终显示 |
| `AI-Powered · v1.0` | 副标题 | 版本标识 |
| `📊 需求看板` | 导航项 | 点击切换到需求看板页（高亮激活状态） |
| `📋 所有需求 ▶` | 导航项 + 折叠箭头 | 点击展开子菜单：大区需求 / SaaS 厂商需求 |
| `🌐 大区管理` | 导航项 | 点击切换到大区管理页 |
| `🏢 SaaS厂商管理` | 导航项 | 点击切换到 SaaS 厂商管理页 |
| `周期：2026-W22 · 批次 #14` | 底部信息 | 显示当前周期和批次，`❓` 点击触发新手引导 |
| `❓` | 图标按钮 | 点击呼出**新手引导弹窗** |

#### 1.2 顶部栏（Topbar）

| 元素 | 类型 | 功能说明 |
|---|---|---|
| `需求看板 · Requirement Dashboard` | 页面标题 | 显示当前页面名称 |
| `全部国家` 下拉框 | 筛选器 | 选项：全部 / 🇹🇭 泰国 / 🇮🇩 印尼 / 🇮🇹 意大利 / 🇰🇭 柬埔寨 / 🇲🇾 马来西亚；切换后触发 `renderTable()` 重绘表格 |
| `全部周期` 下拉框 | 筛选器 | 选项：全部 / 2026-W22（当前）/ 2026-W21 / 2026-W20；切换后触发 `renderTable()` |
| `🔍 搜索需求...` 搜索框 | 搜索框 | 输入关键词实时搜索需求簇（大区表格） |
| `归并状态文字` | 状态提示 | 显示上次归并结果状态（如"已完成 20 个簇"） |
| `⚡ AI 归并` 按钮 | 主要按钮（蓝色） | **点击**：弹出 Thinking 遮罩 → 调用后端 `/api/merge` → 归并相似需求 → 刷新表格；失败时 Toast 提示 |
| `⚙ 评分规则` 按钮 | 次要按钮（透明边框） | **点击**：呼出**评分规则面板**（右侧抽屉） |

> **移动端**：`⚡ 归并` 按钮移到顶部栏外侧，筛选器折叠到 `#mobFilterStrip`。

#### 1.3 统计卡片区（4 张卡片）

| 卡片 | 显示字段 | 数据来源 |
|---|---|---|
| `大区工单总量` | 数值 + 环比变化（绿色/红色） | `/api/stats` |
| `大区需求簇` | 数值 + 压缩率百分比 | `/api/stats` |
| `SaaS工单总量` | 数值 + 环比变化 | `/api/stats` |
| `SaaS需求簇` | 数值 + 压缩率百分比 | `/api/stats` |

#### 1.4 图表区（3 张图表）

| 图表 | 类型 | 说明 |
|---|---|---|
| `来源国分布` | 饼图（Chart.js） | 显示各国需求占比；点击图例可隐藏/显示某国家 |
| `归属层分布` | 环形图（Chart.js） | 显示平台层 vs SaaS 层需求占比 |
| `周期提交量趋势` | 折线图（Chart.js） | 显示最近 N 个周期的需求提交趋势 |

#### 1.5 大区需求簇表格（Top 10）

| 列名 | 说明 |
|---|---|
| `ID` | 需求簇 ID（如 `CL-001`），点击可打开详情抽屉 |
| `需求摘要` | AI 生成的摘要（中文） |
| `归属层` | `平台` / `SaaS`（彩色标签） |
| `业务影响` | `高` / `中` / `低`（红/橙/绿标签） |
| `来源` | `大区` / `SaaS厂商`（蓝色标签） |
| `频次` | 归并工单数量 + 进度条 + 数值 |
| `优先级` | 分数 + 进度条（颜色渐变） |
| `状态` | `待处理` / `已归并` / `排期中` / `开发中` / `已上线`（圆点+文字标签） |

**表格下方提示文字**（动态）：
- 当条目 > 10 条时显示：`共 X 个需求簇，仅展示 Top 10 · 前往「大区需求」查看全部`
- 点击「前往「大区需求」」切换到 `page-all-partner`。

**搜索框**：`🔍 搜索需求...`，输入后实时过滤表格（按摘要 + ID 模糊匹配）。

#### 1.6 需求簇详情抽屉（Drawer）

> 点击表格任意一行打开

| 区域 | 元素 | 功能说明 |
|---|---|---|
| 抽屉头部 | `需求簇详情` 标题 + `✕` 关闭按钮 | 点击 `✕` 关闭抽屉 |
| 元信息区 | `ID` / `创建时间` / `工单数量` / `国家` | 只读显示 |
| AI 摘要区 | `✦ AI 需求摘要` 标题 + 摘要正文 | 显示 AI 归并时生成的需求摘要；**可点击编辑**（contenteditable），编辑后 `✓ 保存摘要` 按钮亮起 |
| 来源工单列表 | 每个工单卡片显示：`工单 ID`、`提交时间`、`原始描述`、`商户`、`国家` | 点击工单卡片 → 打开**工单详情抽屉**（见 1.7）；点击 `🗑 移除` 将工单从簇中移除（二次确认） |
| 状态变更区（抽屉底部） | 状态下拉框 + `更新状态` 按钮 | 下拉框选项：`待处理` / `已归并` / `排期中` / `开发中` / `已上线`；点击 `更新状态` → 调用 API 更新状态 + 刷新时间线 |
| 评审面板（可折叠） | `评审` 标题（点击展开/收起）+ 评审表单 | 展开后显示：`人工标签` 下拉（高价值 / 有效 / 重复 / 无效）+ `优先级调整` 下拉（+10 / +5 / 不变 / -5）+ `✓ 保存评审` 按钮；保存后刷新优先级分数 |
| `🗑 拆散需求簇` 按钮（红色边框） | 危险操作按钮 | 点击 → 弹窗确认 → 确认后将该簇内所有工单状态重置为 `pending`、解除归并关系 |

#### 1.7 工单详情抽屉（Ticket Detail Drawer）

> 从"需求簇详情抽屉"内点击工单卡片打开（多层抽屉）

| 区域 | 元素 | 功能说明 |
|---|---|---|
| 头部 | `工单 ID` 标题 + `✕` 关闭 | |
| `原始描述` | 完整原始文本 | 只读 |
| `商户` / `商户类型` / `业务影响` / `涉及场景` | 字段值 | 只读 |
| `提交时间` / `来源大区` / `归并状态` | 字段值 | 只读 |
| `附件` 区 | 图片网格 / 视频缩略图 | 点击附件 → 打开**灯箱大图/视频播放**；鼠标悬停显示 `🔍 放大` 按钮 |

#### 1.8 AI 归并 Thinking 遮罩

| 元素 | 说明 |
|---|---|
| 旋转加载动画 | 归并进行中显示 |
| `DeepSeek Thinking...` | 标题 |
| 动态子文本 | 显示当前步骤（如"正在分析工单并归并需求簇"） |

#### 1.9 评分规则面板（右侧抽屉）

> 点击顶部栏 `⚙ 评分规则` 打开

| 区域 | 元素 | 功能说明 |
|---|---|---|
| 业务影响差距 | 3 个预设选项：`🟰 均等`（不区分影响等级）/ `⚖️ 适中差距`（默认）/ `🚀 高度强调` | 点击选中，高亮边框 |
| 来源国战略优先级 | 国家排序列表（可**拖拽排序**）| 拖拽调整顺序 → 排名越靠前权重越高；每个国家右侧显示权重加成百分比（自动计算） |
| 大区重要度 | 每个大区一行，右侧有 `战略级` / `重要级` / `普通级` 下拉框 | 切换后自动保存；战略级加权最高 |
| 跨周期累计加成 | 3 个预设选项：`➖ 不加权` / `📈 适中加成`（默认）/ `🔥 显著加成` | 点击选中 |
| `取消` 按钮 | 关闭面板，不保存 | |
| `✦ 重新计算得分` 按钮（蓝色） | 点击 → 按当前规则重新计算所有需求簇的优先级分数 → 刷新表格 | **重要**：此操作会覆盖之前的优先级排序 |

#### 1.10  Toast 提示

| 元素 | 说明 |
|---|---|
| 底部右侧弹出提示 | 归并完成、状态更新、得分重算等操作后显示；2 秒后自动消失 |

#### 1.11 确认弹窗（Confirm Dialog）

| 场景 | 标题 | 内容 | 按钮 |

| 移除工单 | `确认移除` | `确定要将此工单从需求簇中移除吗？` | `取消` + `确认移除` |

#### 1.12 新手引导弹窗（Onboarding）

### 页面 5：需求提报（`需求提报.html`）

> 文件：`需求提报.html`（原 `form.html`）  
> 访问方式：`需求提报.html?token=<大区令牌>`

#### 5.1 语言切换栏（顶部）

| 按钮 | 说明 |
|---|---|
| `EN` / `中文` / `IT` / `FR` / `ไทย` / `ខ្មែរ` | 点击切换表单语言；**表单字段标签 + 占位符 + 按钮文字**全部切换为对应语言 |

#### 5.2 头部信息区

| 元素 | 说明 |
|---|---|
| `🌏 Requirement Collection System` / `需求采集系统`（依语言） | 系统徽章 |
| `Submit Merchant Requirement` / `提交商户需求`（依语言） | 页面标题 |
| `Fill in your local language — we'll handle the translation` / 对应翻译 | 副标题 |
| `🌏 Partner: TH-Partner` | 自动识别令牌对应的大区名称 |
| `Current Cycle: 2026-W22` | 当前周期（自动） |

#### 5.3 表单字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `需求描述` `*` | 多行文本框 | 必填；接受任意语言输入；AI 会自动翻译 |
| `商户` `*` | 单行文本框 | 必填；商户名称或编号 |
| `商户类型` `*` | 单选按钮组 | `🍜 餐厅` / `🛒 零售` / `📦 其他`；必填 |
| `业务影响` `*` | 单选卡片组 | `🔴 高影响`（影响核心流程）/ `🟡 中影响`（影响部分功能）/ `🟢 低影响`（体验改善）；必填 |
| `涉及场景` `*` | 多选复选框组 | `🖥️ POS` / `📊 报表` / `👤 会员` / `📟 KDS` / `💳 支付` / `📦 库存`；至少选一个 |

## 后端 API 接口清单

> 基于 `backend/main.py` 整理

### 认证方式

所有需要认证的接口均在 URL 中传 `?token=xxx` 参数；后端通过 `get_current_partner(token)` 或 `get_current_vendor(token)` 验证令牌有效性。

---

### Tickets（工单）接口

| 方法 | 路径 | 说明 | 请求体 / 参数 | 响应 |
|---|---|---|---|---|
| `POST` | `/api/tickets` | **提交大区工单**（需求提报页调用）| `{ token, text, merchant, impact, scenes[], biz_type, lang }` | `{ id: "TH-0001" }` |
| `GET` | `/api/tickets?token=xxx&page=&page_size=200` | **获取大区工单列表** | 查询参数：`token`, `page`, `page_size`, `country`, `status`, `impact`, `search` | `{ items: [...], total }` |
| `POST` | `/api/saas-tickets` | **提交 SaaS 厂商工单**（SaaS 需求提交页调用）| 同 `/api/tickets`，但 token 前缀为 `SV_` | 同左 |
| `GET` | `/api/saas-tickets?token=xxx&...` | **获取 SaaS 工单列表** | 同 `/api/tickets` | 同左 |

### Clusters（需求簇）接口

| 方法 | 路径 | 说明 | 请求体 / 参数 | 响应 |
|---|---|---|---|---|
| `GET` | `/api/clusters?token=xxx&page_size=200` | **获取大区需求簇列表** | 查询参数：`token`, `country`, `status` | `{ items: [...], total }` |
| `GET` | `/api/saas-clusters?token=xxx&...` | **获取 SaaS 需求簇列表** | 同左 | 同左 |
| `POST` | `/api/merge` | **触发 AI 归并**（看板页 `⚡ AI 归并` 按钮调用）| `{ token }` | `{ merged: 20, clusters: [...] }` |
| `PUT` | `/api/clusters/{id}/status` | **更新需求簇状态** | `{ status: "live" }` | `{ success: true }` |
| `PUT` | `/api/clusters/{id}/dissolve` | **拆散需求簇** | `{}` | `{ success: true }` |
| `PUT` | `/api/clusters/{id}/summary` | **更新 AI 摘要**（编辑摘要后保存）| `{ summary: "..." }` | `{ success: true }` |
| `PUT` | `/api/clusters/{id}/review` | **保存人工评审** | `{ tag: "high_value", priority_delta: +5 }` | `{ success: true }` |

## 端到端业务流程

### 流程 1：大区提交需求 → 归并 → 上线

```
大区人员
  ↓ 打开 `需求提报.html?token=xxx`
  ↓ 填写表单（描述/商户/类型/影响/场景）+ 上传附件
  ↓ 点击"提交需求"
  ↓ 成功 → 显示工单 ID（如 TH-0001）
  ↓ 可"继续提交"或关闭
  
PM（定期）
  ↓ 打开 `需求看板.html`
  ↓ 点击"⚡ AI 归并"
  ↓ 系统调用 DeepSeek API 分析所有 pending 工单
  ↓ 相似工单归并为需求簇（生成 AI 摘要 + 优先级分数）
  ↓ 归并完成后 Toast 提示
  
PM
  ↓ 查看需求簇详情 → 编辑摘要（如需要）→ 保存
  ↓ 设置状态为"排期中"→ 填写排期时间线
  ↓ （开发完成后）设置为"开发中"
  ↓ （上线后）设置为"已上线"
  
大区人员
  ↓ 打开 `我的需求追踪.html?token=xxx`
  ↓ 查看工单状态时间线 → 看到"已上线"
```

### 流程 2：PM 人工录入 / 批量导入

```
PM
  ↓ 打开 `所有需求` → `原始需求` Tab
  ↓ 点击"＋ 人工录入"
  ↓ 填写表单（PM 代大区填写）→ 保存
  ↓ 或点击"⬆ 批量导入"
  ↓ 下载模板（选择语言）→ 填写 Excel → 上传
  ↓ 系统解析 Excel → 批量创建工单
```
